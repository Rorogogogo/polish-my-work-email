## English

> Write clear, natural workplace English — not fancy, not robotic. Built for people whose first language isn't English and who work at Australian companies.

You paste a rough message. The skill gives you a clean, ready-to-send version, asks first when the meaning is risky or unclear, points out anything to check before you send, shows what changed, and saves the result as files for each platform.

**What it does**

- ✅ Rewrites into **simple, natural Australian** workplace English — not too formal, not too casual, not AI-sounding.
- ❓ **Asks first when guessing could change the meaning**, especially around deadlines, blame, ownership, commitments, or technical conclusions.
- ⚠️ **Flags assumptions and risky wording** before you send: hidden deadlines, blame, commitments, ownership, or technical conclusions.
- 💬 **Channel-aware** formatting: Email (Outlook/Gmail), Teams/Slack (RTF rich text), and Markdown for Confluence/docs.
- 📁 **Saves files automatically**, named per platform, so you just copy and paste.
- 🔍 **Unified `.diff`** so you can see exactly what changed, line by line.
- 📸 **Screenshot suggestions** with numbered `[ Screenshot 1 ]` placeholders showing exactly where to insert each image.
- 📊 **Easy-to-scan reply**: the message to send comes first; decisions and changes are shown in small tables.
- 🎓 **Teaches you as you go** with short, simple notes on why a phrase was changed.

## 简体中文

> 帮你把英文工作消息改得清楚又自然。不用写得很高级，也不要像 AI。适合英语不是母语、在澳洲公司工作的人。

你把草稿贴进来，它会先看有没有不能乱猜的地方。意思明确时，它会给你一版可以直接发的英文，提醒你发送前要确认什么，说明改了哪里，并按不同平台保存成文件。

**它能做什么**

- ✅ 把英文工作消息改成**简单自然的澳洲职场表达**。不太正式，不太随便，也不像 AI 写的。

- ⚠️ 发送前提醒你哪些地方需要确认，避免无意中把话说得太满或太硬，也避免听起来像在定责。
- 💬 按使用场景输出：邮件（Outlook/Gmail），Teams/Slack（RTF 富文本），以及适合 Confluence/文档的 Markdown。
- 📁 自动保存成对应平台的文件，打开就能复制粘贴。

- 📸 如果截图能说明问题，会告诉你截图放在哪里，以及该截什么。
- 📊 回复结构清楚：先给可发送版本，再列出需要确认的点和主要改动。

## Install

This is a [Claude](https://claude.com/claude-code) skill. Put the folder where Claude can find it:

```bash
# For all your projects (global)
git clone https://github.com/Rorogogogo/polish-my-work-email.git \
  ~/.claude/skills/polish-my-work-email

# Or just for one project
git clone https://github.com/Rorogogogo/polish-my-work-email.git \
  .claude/skills/polish-my-work-email
```

Then start Claude Code in your project. The skill loads automatically.

## Usage

Just ask in plain language, for example:

- "Polish this email: …"
- "Make this sound natural for Teams: …"
- "Is this too direct to send to my manager? …"

You can also call it directly with `/polish-my-work-email`.

The reply leads with the **ready-to-send message**, lists anything to **check before sending**, shows **what changed**, suggests a **screenshot** if it helps, and saves files to an `output/` folder in your working directory.

## Example

**You paste:**

> Hi Mark, sorry to disturb. As per my understanding the sales dashboard number is wrong, total revenue showing 1.2M but yesterday we discuss it should be 1.5M. I have doubt the data pipeline is broken because last run failed last week. Kindly check and revert by today EOD as board meeting is tomorrow. Please advise.

**The skill replies:**

### ✅ Ready to send (Email)

> Hi Mark,
>
> I noticed the sales dashboard total revenue is showing **$1.2M**. From our chat yesterday, I understood it should be around **$1.5M**.
>
> `[ Screenshot 1 ]`
>
> I also saw the last data pipeline run failed last week, so it might be related. I haven't confirmed that yet.
>
> `[ Screenshot 2 ]`
>
> Since the board meeting is tomorrow, could you take a look today if you have time? Happy to share what I've found.
>
> Thanks, Robert

### ⚠️ Please check before sending

| # | Check this | What I did |
|---|------------|------------|
| 1 | The $1.5M figure | Wrote "from our chat I understood". Your memory, not stated as fact |
| 2 | Pipeline as the cause | Softened to "might be". Confirm before blaming it |
| 3 | "by today EOD" | Changed to "today if you have time". A hard deadline to your manager can sound pushy |

### ✏️ What I changed

| Original | Better | Why |
|----------|--------|-----|
| As per my understanding | from our chat I understood | more natural in AU |
| I have doubt | it might be | "doubt" means you don't believe it. Wrong word here |
| revert by today EOD | take a look today if you have time | "revert" sounds wrong; softer on a manager |

### 📸 Screenshot would help

| # | Where it goes | Capture | Tip | Caption |
|---|---------------|---------|-----|---------|
| 1 | After the revenue numbers | The dashboard showing $1.2M | Circle the number; keep the date filter visible | "Dashboard shows $1.2M for this period" |
| 2 | After the pipeline sentence | The failed data pipeline run | Include the run time and error status | "Last pipeline run failed last week" |
